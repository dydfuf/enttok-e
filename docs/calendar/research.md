# 캘린더 연동 UX 리서치

## 범위
enttokk-e에서 Google Calendar 데이터를 노출하는 UX 베스트프랙티스와 UI 패턴을 정리한다. 범위는 다음과 같다.
- 데일리 노트: 오늘 일정만 노출
- 캘린더 탭: 전체 일정(읽기 전용)

## 확정된 결정
- 데일리 노트에는 오늘 일정만 표시한다(로컬 타임존 기준 00:00-23:59).
- 캘린더 탭은 읽기 전용으로 제공한다.
- 데일리 노트에서 "회의 노트 만들기" 액션이 필요하다.
- 녹음 기능을 제공하는 것이 바람직하다.
- 개인/비공개 일정도 기본적으로 노출한다(마스킹 기본값 아님).

## 베스트프랙티스 요약 (외부 출처)
- 일정과 할 일을 함께 보여주는 "오늘/예정" 뷰가 계획에 유리하며, 캘린더 선택과 숨김이 필요하다. (Todoist)
- 일/주/월 뷰 제공, 이벤트 가독성 최우선. (Nylas, Pageflows)
- 인터페이스 단순성, 날짜 이동 내비게이션 명확성, 반응형이 핵심. (Pageflows)
- 이벤트 맥락에서 상세와 액션을 제공하는 컨텍스트 UI가 효과적. (Google Workspace add-on UI)
- 성능/신뢰성/동기화 안정성을 우선하고 제공자 변경에 대비. (Nylas)
- 타임라인 + 그리드 혼합 등 여러 뷰와 낮은 인지부하가 효과적. (Eleken)

## UX 제안

### 데일리 노트 (오늘 일정만)
- 섹션 타이틀: "오늘" 또는 "오늘 일정"을 데일리 노트 상단에 고정.
- 리스트 정보: 시작/종료 시간, 제목, 캘린더 색상 점. 장소/미팅 링크는 있을 때만.
- 시각 강조:
  - 진행 중 이벤트는 약한 하이라이트.
  - 지난 이벤트는 톤 다운.
  - 예정 이벤트는 기본 톤.
- 종일 일정: 리스트 최상단에 "종일" 라벨로 분리.
- 빈 상태: 짧은 안내 문구 + "캘린더 열기" 버튼.
- 이벤트별 액션:
  - 회의 노트 만들기(제목/시간/참석자 프리필).
  - 미팅 링크 열기(있을 때만).
  - 캘린더에서 보기(딥링크 가능 시).
- 녹음:
  - 이벤트 상세에서 "녹음 시작" 제공.
  - 생성된 회의 노트에 녹음 파일 링크/참조를 저장.
  - 자동 녹음은 지양, 명시적 opt-in만 허용.

### 캘린더 탭 (전체, 읽기 전용)
- 기본 뷰: 주간(일/주/월 토글 제공).
- 좌측 패널: 캘린더 목록(토글/색상).
- 미니 월 캘린더로 빠른 날짜 이동.
- 이벤트 클릭 시 상세 패널(우측 패널 또는 모달).
- 읽기 전용을 명확히 표기(드래그 핸들/인라인 수정 제거).

### 이벤트 상세 패널
- 기본 액션: 회의 노트 만들기, 미팅 링크 열기, 캘린더에서 보기.
- 보조 액션: 일정 정보 복사, 미팅 링크 복사.

### 상태 및 신뢰성
- 로딩 상태: 스켈레톤 + 마지막 동기화 시간 표시.
- 오류 상태: 재시도 버튼 + 캐시된 데이터로 폴백.
- 오프라인: 마지막 데이터 표시, "오래됨" 배지 표기.

## 데이터/프라이버시/성능 메모
- 일정 데이터는 가능한 한 복제 저장하지 않고 참조만 유지. (Todoist)
- API 호출 최소화(캐시 + 증분 동기화), 느린 로딩이 UX를 해치지 않도록 설계. (Nylas)
- 스코프는 최소 권한 원칙(읽기 전용이 기본, 쓰기는 필요한 기능에 한정).
